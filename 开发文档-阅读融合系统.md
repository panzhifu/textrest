# Legado + 本地阅读融合系统开发文档

> 面向单人开发的 Flutter + Rust 阅读系统。本文档用于指导模块划分、接口设计与迭代计划。

---

## 1. 项目概述

**目标**：构建一款阅读软件，融合 Legado 书源规则解析与本地书库阅读体验，实现统一搜索、统一书架与阅读器。

**技术栈**：
- 前端：Flutter（多端 UI）
- 后端：Rust（书源引擎、网络请求、数据库、规则执行）
- 数据库：SQLite（本地）

**MVP 关键词**：书源导入、统一搜索、本地导入、阅读器、阅读记录

---

## 2. 功能范围

### 2.1 MVP（第一阶段）
1. 书源导入与管理（Legado JSON）
2. 书源搜索
3. 本地书籍导入（txt/epub）
4. 统一书架
5. 章节阅读 + 记录

### 2.2 后续扩展
- 书源评分/校验
- 智能切源
- 离线缓存管理
- 书籍同步/云备份
- 听书与朗读

---

## 3. 系统模块划分

### 3.1 Rust 核心模块

#### A. BookSource Engine（书源引擎）
- 解析 Legado JSON 规则
- 统一书籍结构输出
- 支持搜索/详情/章节/正文

#### B. Network + Cache（网络与缓存）
- 请求封装（重试/超时/UA）
- 搜索、章节缓存
- 离线内容保存

#### C. Local Library（本地书库）
- 本地文件导入（txt/epub）
- 章节切分与索引
- 本地搜索

#### D. Data Layer（数据库）
- 表：books、chapters、sources、records
- 读写记录、书架状态
- 书源启用/排序

#### E. API 层（Flutter 调用）
- FFI 调用（推荐）
- 统一数据结构输出
- 异步任务队列

---

### 3.2 Flutter UI 模块

#### A. Library（书架）
- 本地 + 网络书籍统一展示
- 最近阅读/分组

#### B. Search（搜索）
- 统一搜索入口
- 分来源展示

#### C. Reader（阅读器）
- 章节缓存
- 翻页/滚动
- 夜间模式

#### D. Settings（设置）
- 书源管理
- 缓存清理
- 本地导入

---

## 4. 关键交互流程

### 4.1 统一搜索流程
1. Flutter 发送搜索请求
2. Rust 同时查询本地 + 书源
3. 返回统一 Book 列表

### 4.2 获取章节
1. 点击书籍
2. Rust 根据来源解析章节列表
3. Flutter 展示章节

### 4.3 阅读流程
1. 请求章节内容
2. Rust 解析、缓存
3. Flutter 阅读 + 记录进度

## 6. 接口设计（建议）

### 6.1 搜索
- `search_books(query: String) -> Vec<Book>`

### 6.2 书籍详情
- `get_book_info(book_id: String) -> Book`

### 6.3 章节列表
- `get_chapters(book_id: String) -> Vec<Chapter>`

### 6.4 章节内容
- `get_chapter_content(book_id: String, index: i32) -> Chapter`

### 6.5 本地导入
- `import_local_book(path: String) -> Book`

## 数据模型设计

# 数据结构汇总（含书源与规则）

> 说明：以下为当前项目 `rust/src/models/` 中的完整数据结构汇总。书源与规则结构保持不变，仅整理输出，并补充阅读提示。

## 通用说明

- 时间类字段目前统一为 `String`（例如 `add_time`、`last_read_time`、`last_update_time`、`update_time`）。具体格式由前后端约定（建议 ISO8601）。
- 书源规则中大量字段为 `Option<String>`，表示该规则字段可选，未配置时为 `None`。
- `#[serde(alias = "...")]` 表示兼容字段别名（历史/第三方规则格式）。
